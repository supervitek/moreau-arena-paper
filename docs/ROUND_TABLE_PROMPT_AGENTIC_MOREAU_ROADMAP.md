# Round Table Prompt — Agentic Moreau Roadmap Review

Use this prompt for the next Round Table review.

---

We have produced a draft roadmap for a possible strategic expansion of Moreau Arena toward agentic participation.

Please read this roadmap first:

`docs/AGENTIC_MOREAU_ROADMAP_DRAFT.md`

Context:
- Moreau Arena currently has a benchmark/research side and a game/world side
- We are considering a third layer: agentic participation
- We do NOT want to become a generic AI platform
- We want a serious critique, not encouragement

Please evaluate the roadmap as if advising a small, serious project with limited bandwidth and strong taste.

Focus on these questions:

1. Is the roadmap strategically coherent, or is it still trying to be two incompatible things at once?
2. Is the recommended sequence correct:
   - substrate first
   - internal runtime
   - visible flawed advisory agent first
   - narrow trusted external pilot later
3. Is the recommended first prototype correct:
   - `Chronicler` as a diegetic, advisory, visibly fallible agent
   - not a public BYO-agent API
   - not a hosted champion
4. What is the single biggest mistake in the roadmap?
5. What is the single strongest part of the roadmap?
6. Which phase is most likely to fail in practice?
7. What should be cut, simplified, or delayed?
8. What must be added before execution begins?
9. How should we preserve Moreau's identity, weirdness, and human interpretive role while adding agents?
10. If you had to choose one of these first steps, which would you choose and why?
   - A. Chronicler advisory prototype
   - B. hidden internal stagehand only
   - C. narrow BYO-agent pilot
   - D. no agent pivot at all

Please structure your answer as:

## Verdict
- expansion or dilution?
- go / cautious go / no-go

## Main Critique
- 3 strongest objections

## Main Endorsements
- 3 strongest reasons the roadmap is sound

## First Prototype Recommendation
- pick A / B / C / D
- explain why

## Phase Review
- assess Phase 1 through Phase 5

## What To Cut
- list anything overbuilt or premature

## What To Add
- list missing constraints, metrics, or product decisions

## Final Recommendation
- one-paragraph advice to the team

Important constraints:
- do not recommend turning Moreau into a generic platform
- do not assume a large team
- do not assume enterprise sales
- keep the narrative layer load-bearing
- assume the team needs leverage, not scope explosion

We especially care about sharp disagreement and failure analysis.
